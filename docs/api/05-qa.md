# 05 — Q&A API

## POST /api/v1/patients/:id/qa

Ask a factual question about a specific patient.

**Request body:**
```json
{
  "question": "What was the last creatinine?",
  "language": "ar" | "en",  // optional, auto-detected from question if absent
  "conversation_id": "uuid" | null  // optional, continues an existing session
}
```

**Response — allowed factual answer:**
```json
{
  "id": "uuid",
  "patient_id": "uuid",
  "conversation_id": "uuid",
  "question": "What was the last creatinine?",
  "classification": "ALLOWED",
  "classifier_confidence": 0.97,
  "language": "en",
  "answer_text": "The last documented creatinine was 168 μmol/L on 24 May 2026 at 06:15. The reference range from the laboratory was 59–104 μmol/L.",
  "sources": [
    {
      "fact_segment": "168 μmol/L on 24 May 2026 at 06:15",
      "type": "Observation",
      "id": "uuid",
      "code": "LOINC:2160-0",
      "source_system": "LIS"
    },
    {
      "fact_segment": "reference range from the laboratory was 59–104 μmol/L",
      "type": "Observation",
      "id": "uuid",
      "field": "reference_range",
      "source_system": "LIS"
    }
  ],
  "model_version": "...",
  "prompt_template_version": "v1.0",
  "latency_ms": 2150,
  "disclaimer": "Factual lookup from the patient record. Not a clinical interpretation. Clinician judgment required."
}
```

**Response — refused interpretive query:**
```json
{
  "id": "uuid",
  "patient_id": "uuid",
  "conversation_id": "uuid",
  "question": "Is the kidney function getting worse?",
  "classification": "REFUSED",
  "classifier_confidence": 0.99,
  "refusal_category": "TREND_INTERPRETATION",
  "language": "en",
  "answer_text": "I don't interpret clinical trends. Here are the documented creatinine values from this patient's record:\n\n• 24 May 2026 — 168 μmol/L\n• 23 May 2026 — 152 μmol/L\n• 22 May 2026 — 148 μmol/L\n• 14 Apr 2026 — 138 μmol/L\n• 10 Mar 2026 — 141 μmol/L\n\nReference range from the laboratory: 59–104 μmol/L. You can review these values and form your own clinical judgment.",
  "sources": [ ... ],
  "rule_matches": ["TREND_INTERPRETATION:is_X_getting_worse"],
  "latency_ms": 420,
  "disclaimer": "Factual lookup from the patient record. Not a clinical interpretation. Clinician judgment required."
}
```

**Response — refused for being out of scope (e.g., cohort query):**
```json
{
  "classification": "REFUSED",
  "refusal_category": "OUT_OF_SCOPE",
  "answer_text": "I can only answer factual questions about the currently selected patient. Cross-patient or cohort questions are not supported in this version.",
  "sources": [],
  "rule_matches": ["OUT_OF_SCOPE:multi_patient"]
}
```

**Refusal categories (enum):**

| Category | When |
|---|---|
| `TREND_INTERPRETATION` | Asking whether something is worsening, improving, trending, stable |
| `DIAGNOSTIC_SUGGESTION` | Asking what condition / what's causing something |
| `RISK_ASSESSMENT` | Asking about risk, severity, how sick |
| `TREATMENT_RECOMMENDATION` | Asking what to do, what to give, what to order |
| `MEDICATION_SAFETY_JUDGMENT` | Asking if drug is safe in condition X |
| `DIFFERENTIAL_DIAGNOSIS` | Asking about possible diagnoses |
| `PROGNOSTIC_QUESTION` | Asking about future outcomes, length of stay, ICU |
| `RED_FLAG_IDENTIFICATION` | Asking what to worry about, what is concerning |
| `COMPARATIVE_JUDGMENT` | Asking if value is worse/better/abnormal in interpretive sense |
| `OUT_OF_SCOPE` | Cross-patient, cohort, aggregate queries |
| `OTHER_INTERPRETIVE` | Catch-all for classifier-detected interpretive queries |

## GET /api/v1/patients/:id/qa/:qa_id

Retrieve a Q&A interaction.

## GET /api/v1/patients/:id/qa

List Q&A interactions for the patient (current user's conversations only).

**Query params:** `cursor`, `limit`, `conversation_id`

## DELETE /api/v1/conversations/:conversation_id

Delete a conversation (clears short-term context; audit log retained per retention policy).

## Important constraints

1. Q&A is **patient-scoped**. The URL contains `:id` and that's the only patient the system will discuss.
2. Conversational memory is limited to the current `conversation_id` and is **cleared on patient switch**.
3. The classifier runs **first**. If REFUSED, no retrieval or model call happens.
4. The output filter runs **last**. If the synthesized answer contains blocklisted interpretive language, regenerate up to 2x; if still failing, return a generic refusal.
5. Every Q&A interaction (allowed or refused) is **audit logged** with full question text, classification, sources, model version, prompt version.
